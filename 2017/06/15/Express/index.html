#二、Express

  1. 什么是express？
    • Express 是基于node.js 平台、快速、开发、极简的一个’’ web 开发框架’’
    • 同时也是‘Node.js’的一个第三方模块
  2. 为什么学习express框架?
    • 让我们基于node.js 开发 web 应用程序更高效
  3. express官方网站
    • 英文官网-http://expressjs.com
    • 中文官网-http://www.expressjs.com.cn/

##Express特点

  1. 实现了 路由功能（eg:express.Router()）
  1. 中间件（函数）功能:(eg:app.use(‘/‘,function(){…}))、配合use一块使用
  2. 对req和res对象的拓展
  3. 可以集成其他模版引擎（eg:ejs）

##Express 基本使用

  1. 基本步骤

    • 安装 npm i express // 安装之前 先看下有没有 package.json文件 没有 npm init -y
    • 加载； let express = require('express')
    • 实例： let app = express() 实例化express对象
    • 使用：app.get()/app.use()/app.all() / app.listen()

  2. 案例演示

  1. res.send()和res.end()的异同

    • 相同点 都能结束响应，把内容响应给浏览器

    • 不同点：

      ​ 1.res.send()不乱码

      ​ 2.参数类型不同：

      • 在express推荐 res.send()

    4.res 的其他几个常用的方法

    • res.redirect([status,]path) ：重定向
  • res.sendFile(path [, options]_[, fn]) 读取文件
  • res.statis():设置状态

Express 中注册路由的方法

方法一：app.METHOD()

类型固定，路径完全匹配

  1. 基本用法
- Method 是一个http请求方法：get/post/put/delete等等

-

方法二： app.use() — (中间件一般与这个配合使用)

开头是: /index 就匹配

任意类型,路径开始相同就匹配

    1. 在进行路由匹配的时候,不限定方法,什么请求方法都可以
    1. 请求路径的第一部分只要与 /index 相等即可,,,并不要求请求路径 ( pathname ) 完全匹配

方法三： app.all()

任意类型，路径完全匹配

  1. 不限定请求方法

  2. 请求路径的pathname 必须完全匹配

Express处理静态资源

额外补充:

  1. 例如:静态文件叫: demo.css
    • ./demo.css —> 请求路径的第一个部分是: /
    • ./public/demo.css —> 请求路径的第一个部分是: /public
  2. express 中的 req.url 已经不是之前的那个了
    • /public/demo.css —> http的 req.url = /public/demo.css
    • /public/demo.css —> express的 req.url = /demo.css
